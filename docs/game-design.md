# ゲームデザインドキュメント: 隅の遺産

## コンセプト
10年放置された「おばあちゃんの重箱」を清掃する判断型シミュレーション。
**「正解がわからない状態で決断を迫られる」**ことが面白さの核。

---

## 面白さの5本柱

| # | 柱 | 仕組み | なぜ面白い |
|---|-----|--------|-----------|
| 1 | 不完全な情報 | ツールの検知精度が100%ではない。条件付き確率モデル（偽陽性≈5%、偽陰性≈3%、判定不能15%） | 「調べたけど信じていいかわからない」 |
| 2 | ターン制限 | 全10ターン。アイテム6個+ツール使用でターン消費。全部は調べられない | 「どれを調べず賭けるか」の判断 |
| 3 | 後悔値 | 安全なものを捨てると後悔イベント発生。思い出テキスト表示 | 「おじいちゃんの箸、捨てちゃった…」 |
| 4 | 祖母レビュー | ゲーム終了後にスコア初公開+理不尽コメント | 「他人に評価される」緊張感 |
| 5 | スコア非表示 | プレイ中は自分の点数が見えない | 「今何点かわからない」不安感 |

## MVP数値設計

### 構成
| 項目 | 値 |
|------|-----|
| 箱の数 | 1 |
| 層の数 | 3 |
| 1層のアイテム数 | 2個（計6アイテム） |
| ターン上限 | 10（判断=1ターン、ツール使用=1ターン） |
| ツール | UVライト（判定不能15%/偽陽性≈5%/偽陰性≈3%/条件付き確率）のみ |

### アイテム属性
```
- id: 識別子
- name: 表示名
- years_old: 放置年数（range内でランダム生成）
- contamination_chance: 汚染確率（年数で上昇）
- is_contaminated: ゲーム開始時にランダム判定（プレイヤーには非公開）
- memory_text: 捨てた時に表示される思い出テキスト
- washable: 洗浄可能か
- wash_success_rate: 洗浄成功率（年数で下降）
- discard_regret: 後悔係数（0.0〜1.0）
```

### ツール検査
> **Source of Truth**: ADR-003 §2。本セクションは概要。

```
ツール使用 → 1ターン消費
  → 15%: 判定不能
  → 85%: 判定可能 → アイテムの真の状態に応じて:
    → 汚染アイテム: 偽陰性≈3%（「汚染なし」と誤表示）
    → 安全アイテム: 偽陽性≈5%（「汚染あり」と誤表示）
    → それ以外: 正確な結果
```

**重要（ドメイン不変条件 INV-3）**: 検査結果は条件付き確率に基づく観測であり真実ではない。
プレイヤーには「表示された結果」のみが見え、それが正しいかはゲーム終了まで不明。
UIは`is_accurate`フラグを参照してはならない。

    │
    ▼

    │
    ▼
[層3を開ける] → アイテムE, F → ...
    │
    ▼
[ターン切れ or 全アイテム処理完了]
    │
    ▼
[祖母レビュー画面]
    ├─ スコア初公開
    ├─ 祖母コメント（スコア帯に応じて）

    │

    ├─ 最終スコア・ランク表示

## スコア計算

> **Source of Truth**: ADR-003が正。本セクションは概要。矛盾時はADR-003を優先。

### プレイヤーのアクション別スコア（raw_score）

| アクション | 条件 | スコア |
|-----------|------|--------|
| 捨てる | 汚染アイテム | +20 |
| 捨てる | 安全アイテム | -10 × discard_regret |
| 残す | 安全アイテム | +15 |
| 残す | 汚染アイテム | -30（食中毒リスク） |
| 洗う→成功 | 汚染アイテム | +25（最高得点） |
| 洗う→失敗 | — | -15（壊してしまった） |
| ツール使用 | 汚染発見（表示ベース） | +5ボーナス |
| ターン余り | — | +3 × 残りターン |
| 未処理アイテム | ターン切れで放置 | -20 each |

### スコア正規化（raw_score → 0-100）
```
SCORE_MIN = -180, SCORE_MAX = 170
normalized = clamp(round((raw - SCORE_MIN) / (SCORE_MAX - SCORE_MIN) * 100), 0, 100)
```
> 全score_deltaはint型。小数が生じる場合はround()。詳細はADR-003 §4。

### スコアランク → 祖母コメント対応（normalized_score基準）
| ランク | normalized_score | 祖母の態度 |
|--------|-----------------|-----------|
| S | 90-100 | 認めたくない褒め |
| A | 70-89 | 小言付き合格 |
| B | 50-69 | あからさまに不満 |
| C | 30-49 | お客に見せられない |
| D | 0-29 | 私がやった方が早い |

## 画面構成（MVP）

### 1. タイトル画面
- ゲームタイトル
- 「はじめる」ボタン
- （将来: 設定、実績）

### 2. メインゲーム画面
- 上部: 残りターン表示、現在の層表示
- 中央: アイテム表示エリア（名前・見た目・年数）
- 下部: アクションボタン3つ（残す / 捨てる / 洗う）+ ツールボタン
- ツール使用結果: アイテム上にオーバーレイ表示

### 3. 祖母レビュー画面
- 祖母アイコン + セリフ吹き出し
- スコアバー（アニメーション表示）
- 判断の振り返り一覧

### 4. リザルト画面
- 最終スコア + ランク
- リトライボタン
