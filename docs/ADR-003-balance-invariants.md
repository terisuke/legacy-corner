# ADR-003: バランス不変条件と数値レンジ

## ステータス: 提案中

## コンテキスト
Codexレビューで以下が指摘された:
1. スコアが0-100に収まらない（加減算の合計が範囲外になる）
2. 汚染確率 `base_chance + years_old * 0.02` が1.0を超える
3. GDD/README/items.jsonで数値が不一致（仕様ドリフト）

本ADRでバランスに関わる全数値を固定し、唯一の正とする。

## 数値の正の序列（Source of Truth）
```
ADR-003 > data/*.json > docs/game-design.md > README.md
```
矛盾がある場合、左が勝つ。

## 1. 汚染確率

### 計算式
```
raw_chance = base_contamination_chance + (years_old * 0.015)
final_chance = clamp(raw_chance, 0.05, 0.85)
```

### 境界条件
| パラメータ | 最小 | 最大 | 理由 |
|-----------|------|------|------|
| base_contamination_chance | 0.05 | 0.7 | JSONテンプレート定義（現行データ実測値） |
| years_old | 1 | 25 | JSONテンプレートのrange（現行データ実測値） |
| final_chance | 0.05 | 0.85 | **0.85上限**: 確定汚染を防ぎ読み合いを維持 |

### 検証: 既存アイテムの最悪ケース（2026-03-20 items.json準拠）
| アイテム | base | max_years | raw_chance | clamped |
|---------|------|-----------|------------|---------|
| 重箱の蓋 | 0.3 | 15 | 0.525 | 0.525 ✅ |
| 漆塗り箸 | 0.4 | 25 | 0.775 | 0.775 ✅ |
| 正体不明の布 | 0.55 | 20 | 0.85 | 0.85 ✅（ちょうど上限） |
| 古いポチ袋 | 0.1 | 10 | 0.25 | 0.25 ✅ |
| 九谷焼の小皿 | 0.25 | 20 | 0.55 | 0.55 ✅ |
| 干からびた葉っぱ | 0.7 | 8 | 0.82 | 0.82 ✅ |

## 2. ツール検査精度

### MVPで使用するツール
UVライトのみ。嗅覚センサー・AI菌解析キットはMVP後。

### 検査結果の確率分布（条件付き確率モデル）

> **重要（Codexレビュー2回目で修正）**: 偽陽性/偽陰性は**アイテムの真の状態を条件とした確率**として定義する。
> 旧実装は単一rollで全パターンを分岐しており、汚染アイテムに偽陽性が適用される矛盾があった。

```
use_tool(item, uv_light, rng):
  # Step 1: 判定可能かどうか（無条件）
  if rng.randf() < 0.15:
    return {displayed_result: "inconclusive", is_accurate: false}

  # Step 2: 判定可能（85%の場合） → 真の状態に応じて誤差率を適用
  if item.is_contaminated:
    # 真実: 汚染 → 偽陰性リスク
    if rng.randf() < 0.035:  # ≈3% overall (3.5% of 85% conclusive)
      return {displayed_result: "clean", is_accurate: false}      # 偽陰性
    else:
      return {displayed_result: "contaminated", is_accurate: true} # 正確
  else:
    # 真実: 安全 → 偽陽性リスク
    if rng.randf() < 0.059:  # ≈5% overall (5.9% of 85% conclusive)
      return {displayed_result: "contaminated", is_accurate: false} # 偽陽性
    else:
      return {displayed_result: "clean", is_accurate: true}         # 正確
```

### 全体の期待確率（アイテムの真の状態で条件分岐後）

| アイテムの真の状態 | 結果 | 全体確率 | 説明 |
|-------------------|------|---------|------|
| 汚染 | 判定不能 | 15% | 無条件 |
| 汚染 | 正しく「汚染あり」 | ~82% | 85% × 96.5% |
| 汚染 | **偽陰性**「汚染なし」 | ~3% | 85% × 3.5% |
| 安全 | 判定不能 | 15% | 無条件 |
| 安全 | 正しく「汚染なし」 | ~80% | 85% × 94.1% |
| 安全 | **偽陽性**「汚染あり」 | ~5% | 85% × 5.9% |

## 3. 洗浄成功率

### 計算式
```
raw_wash_rate = 0.9 - (years_old * 0.025)
wash_success_rate = clamp(raw_wash_rate, 0.1, 0.9)
```

### 条件
- `washable == false` のアイテムは洗浄不可（UIでボタンdisable）
- 洗浄成功 → 汚染除去（安全化）
- 洗浄失敗 → アイテム破損（スコア-15）

## 4. スコア計算

### アクション別スコア

#### 整数ポリシー（Codexレビュー2回目で追加）
> **全score_deltaはint型**。小数が生じる計算はround()で丸める。
> `score_delta = int(round(-10.0 * discard_regret))`
> 例: -10 × 0.95 = round(-9.5) = **-10**、-10 × 0.05 = round(-0.5) = **0**

| アクション | 条件 | スコア（int） |
|-----------|------|--------|
| 捨てる | 汚染アイテム | +20 |
| 捨てる | 安全アイテム | `round(-10 × discard_regret)` |
| 残す | 安全アイテム | +15 |
| 残す | 汚染アイテム | -30 |
| 洗う→成功 | 汚染アイテム | +25 |
| 洗う→失敗 | — | -15 |
| ツール使用 | 汚染発見（表示ベース） | +5 |
| ターン余り | — | +3 × 残りターン |
| 未処理 | ターン切れ | -20 each |

### スコア正規化
```
raw_score = sum(all_score_deltas)   # int
normalized_score = clamp(round((raw_score - SCORE_MIN) / (SCORE_MAX - SCORE_MIN) * 100), 0, 100)
```

### 理論値レンジ（6アイテム、10ターン ← ターン制約を正しく反映）

**ターン制約**: 6アイテムの判断で最低6ターン消費。残り4ターンはツール使用 or 余剰。
ツール使用N回 + 判断6回 = N+6ターン。N ≤ 4。ターン余り = 10 - (N+6) = 4-N。

| シナリオ | 計算 | raw_score |
|---------|------|-----------|
| 最高: 全汚染+全洗浄成功+4ツール発見+0余り | 6×25 + 4×5 + 0×3 | **170** |
| 次善: 全汚染+全洗浄成功+0ツール+4余り | 6×25 + 0×5 + 4×3 | 162 |
| 最低: 全汚染見逃し+4ツール不能+0余り | 6×(-30) + 0 + 0 | **-180** |
| 最低(別パターン): 6見逃し+0ツール+4余り | 6×(-30) + 4×3 | -168 |

```
SCORE_MIN = -180
SCORE_MAX = 170
```

> **注**: 旧SCORE_MAX=192は「6ツール+6判断+4余り=16ターン」を前提としており、
> ターン上限10を超過していた。Codexレビュー2回目（2026-03-20）で修正。

### ランク判定（normalized_score基準）
| ランク | normalized_score | 祖母の態度 |
|--------|-----------------|-----------|
| S | 90-100 | 認めたくない褒め |
| A | 70-89 | 小言付き合格 |
| B | 50-69 | あからさまに不満 |
| C | 30-49 | お客に見せられない |
| D | 0-29 | 私がやった方が早い |

## 5. 受け入れシナリオ（固定seed）

### Scenario A: 高汚染・高後悔（seed=42）
- 期待: 6アイテム中4個が汚染
- 高regretアイテム（漆塗り箸 regret=0.95）が安全
- プレイヤーがツールなしで全部捨てると: 汚染4×20 + 安全2×(-10×regret) = 高スコアだが後悔大
- 検証ポイント: 後悔イベント発火、スコア計算正確性

### Scenario B: 低汚染・高誤検知（seed=123）
- 期待: 6アイテム中1個のみ汚染
- UVライトが偽陽性を返す
- プレイヤーがツール結果を信じて安全アイテムを捨てる → 低スコア
- 検証ポイント: 偽陽性の発生、INV-3（is_accurateがUIに漏れない）

### Scenario C: ターン不足（seed=256）
- 全アイテムにツール使用 → 6ターン消費 → 残り4ターンで6判断は不可能 → 2個未処理
- 検証ポイント: INV-4（ターン0で操作拒否）、未処理ペナルティ
