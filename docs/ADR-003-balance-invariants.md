# ADR-003: バランス不変条件と数値レンジ

## ステータス: 提案中

## コンテキスト
Codexレビューで以下が指摘された:
1. スコアが0-100に収まらない（加減算の合計が範囲外になる）
2. 汚染確率 `base_chance + years_old * 0.02` が1.0を超える
3. GDD/README/items.jsonで数値が不一致（仕様ドリフト）

本ADRでバランスに関わる全数値を固定し、唯一の正とする。

## 数値の正の序列（Source of Truth）
```
ADR-003 > data/*.json > docs/game-design.md > README.md
```
矛盾がある場合、左が勝つ。

## 1. 汚染確率

### 計算式
```
raw_chance = base_contamination_chance + (years_old * 0.015)
final_chance = clamp(raw_chance, 0.05, 0.85)
```

### 境界条件
| パラメータ | 最小 | 最大 | 理由 |
|-----------|------|------|------|
| base_contamination_chance | 0.05 | 0.9 | JSONテンプレート定義 |
| years_old | 1 | 30 | JSONテンプレートのrange |
| final_chance | 0.05 | 0.85 | **0.85上限**: 確定汚染を防ぎ読み合いを維持 |

### 検証: 既存アイテムの最悪ケース
| アイテム | base | max_years | raw_chance | clamped |
|---------|------|-----------|------------|---------|
| 重箱の蓋 | 0.3 | 15 | 0.525 | 0.525 ✅ |
| 漆塗り箸 | 0.5 | 30 | 0.95 | **0.85** ← clamp |
| 正体不明の布 | 0.9 | 20 | 1.20 | **0.85** ← clamp |
| 古いポチ袋 | 0.1 | 10 | 0.25 | 0.25 ✅ |

## 2. ツール検査精度

### MVPで使用するツール
UVライトのみ。嗅覚センサー・AI菌解析キットはMVP後。

### 検査結果の確率分布
```
use_tool(item, uv_light, rng):
  roll = rng.randf()
  if roll > 0.85:  # 15%: 検査失敗
    return "inconclusive"
  elif roll > 0.08:  # 77%: 正確な結果
    return actual_contamination_status
  elif roll > 0.03:  # 5%: 偽陽性（安全→「汚染あり」）
    return "contaminated"  # 実際はclean
  else:              # 3%: 偽陰性（汚染→「汚染なし」）
    return "clean"         # 実際はcontaminated
```

| 結果 | 確率 | プレイヤー表示 |
|------|------|---------------|
| 正確 | 77% | 真の状態 |
| 偽陽性 | 5% | 「汚染あり」（実際は安全） |
| 偽陰性 | 3% | 「汚染なし」（実際は汚染） |
| 判定不能 | 15% | 「判定不能」 |

## 3. 洗浄成功率

### 計算式
```
raw_wash_rate = 0.9 - (years_old * 0.025)
wash_success_rate = clamp(raw_wash_rate, 0.1, 0.9)
```

### 条件
- `washable == false` のアイテムは洗浄不可（UIでボタンdisable）
- 洗浄成功 → 汚染除去（安全化）
- 洗浄失敗 → アイテム破損（スコア-15）

## 4. スコア計算

### アクション別スコア（変更なし）
| アクション | 条件 | スコア |
|-----------|------|--------|
| 捨てる | 汚染アイテム | +20 |
| 捨てる | 安全アイテム | -10 × discard_regret |
| 残す | 安全アイテム | +15 |
| 残す | 汚染アイテム | -30 |
| 洗う→成功 | 汚染アイテム | +25 |
| 洗う→失敗 | — | -15 |
| ツール使用 | 汚染発見（表示ベース） | +5 |
| ターン余り | — | +3 × 残りターン |
| 未処理 | ターン切れ | -20 each |

### スコア正規化（新規追加）
```
raw_score = sum(all_score_deltas)
normalized_score = clamp(round((raw_score - SCORE_MIN) / (SCORE_MAX - SCORE_MIN) * 100), 0, 100)
```

### 理論値レンジ（6アイテム、10ターン）
| シナリオ | 計算 | raw_score |
|---------|------|-----------|
| 最高: 全汚染+全洗浄成功+全ツール発見+4ターン余り | 6×25 + 6×5 + 4×3 | 192 |
| 最低: 全汚染見逃し+未処理なし | 6×(-30) | -180 |
| 実質最低: 3未処理+3汚染見逃し | 3×(-20) + 3×(-30) | -150 |

```
SCORE_MIN = -180
SCORE_MAX = 192
```

### ランク判定（normalized_score基準）
| ランク | normalized_score | 祖母の態度 |
|--------|-----------------|-----------|
| S | 90-100 | 認めたくない褒め |
| A | 70-89 | 小言付き合格 |
| B | 50-69 | あからさまに不満 |
| C | 30-49 | お客に見せられない |
| D | 0-29 | 私がやった方が早い |

## 5. 受け入れシナリオ（固定seed）

### Scenario A: 高汚染・高後悔（seed=42）
- 期待: 6アイテム中4個が汚染
- 高regretアイテム（漆塗り箸 regret=0.95）が安全
- プレイヤーがツールなしで全部捨てると: 汚染4×20 + 安全2×(-10×regret) = 高スコアだが後悔大
- 検証ポイント: 後悔イベント発火、スコア計算正確性

### Scenario B: 低汚染・高誤検知（seed=123）
- 期待: 6アイテム中1個のみ汚染
- UVライトが偽陽性を返す
- プレイヤーがツール結果を信じて安全アイテムを捨てる → 低スコア
- 検証ポイント: 偽陽性の発生、INV-3（is_accurateがUIに漏れない）

### Scenario C: ターン不足（seed=256）
- 全アイテムにツール使用 → 6ターン消費 → 残り4ターンで6判断は不可能 → 2個未処理
- 検証ポイント: INV-4（ターン0で操作拒否）、未処理ペナルティ
